Makefiles

You are currently in the GAMMA make directory. This directory contains files that are Makefile components, i.e. files that are included in other Makefiles.

There are essentially two GAMMA Makefiles, one that is for all of GAMMA and one that is for a single GAMMA module. Thse are the main GAMMA Makefile and the GAMMA Module Makefile. The former is named Makefile and resides in the main GAMMA version directory, one level back from this directory. Oddly enough, the latter is also called Makefile, an it exists in each GAMMA module's source directory. That is, for module "Module" there will be a file called ../src/Module/Makefile. No matter what GAMMA module, its makefile is the identical to other Makefiles.

Why do we need this directory then? Because GAMMA Makefiles are currently required to do many many things. Some of these are only for administrative purposes and not seen by the average user. The end result is that any one makefile has become too large and complicated to easily figure out. To simpify things, each Makefile just contains other makefile "segments" which handle the different tasks needed. The result is smaller makefiles that are modular and easier to both handle and comprehend.

Main GAMMA Makefile

This is a simple shell script to accomplish the tasks needed to prepare for GAMMA configurationand build. The script is used to set the GAMMA version to be inherent in the files (configuration and documentation) where it matters. It also generates the configuration script configure from the precursor files configure.ac, aclocal.m4, and acsite.m4 by invoking the GNU program autoconf.
