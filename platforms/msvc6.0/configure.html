MSVC++
Configuration
(Γ Version 4.1.0)
 
 

This directory exists in support of using Microsoft Visual C++ to build GAMMA and GAMMA based programs on Windows/Intel based computers. You MUST have a working version of MSVC++ on your computer system (PC) before these files will be of any use to you.

In order to successfully use the workspaces provided with GAMMA there are three essential details which you must attend to:

  1. Use a recent version of MSVC++.
  2. Tell MSVC++ that .cc files are C++ files.
  3. Set up the global MSVC++ environment for GAMMA.

MSVC++ Version

 GAMMA code is (mostly) ANSI compliant, so to build and use GAMMA one should have an ANSI compatible C++ compiler. Microsoft Visual C++ Version 5 and beyond is (mostly) ANSI compliant also. As far as I know, MSVC++ versions prior to 5 did NOT have full ANSI compatiblity (but I am not completely certain). GAMMA has been built using MSVC++ 5.0, 6.0, and 7.0. Of course, it is impossible to keep up will all versions of MSVC with all versions of GAMMA.

One should use a minimum of MSVC++ 5.0, probably better with 6.0 and beyond. At the time of this writing, the author is using MSVC++ 6.0 and working with GAMMA 4.1.0. Make sure that the latest service packs have been installed. This is just a good idea in general, but particularly important for version 5.0. The service packs can be obtained from the Microsoft WWW site. These are usually very large so you need a decent internet connection. Here is one such site where you can get the latest service pack for 6.0:

MSVC++ 6 Service Packs

Please contact the GAMMA administrators if you believe there are MSVC++ compatibility problems with the GAMMA code.

Setting .cc As C++ Files

 A big issue for GAMMA builds under MSVC++ is use of .cc versus .cpp file suffixes when designating a file of C++ code. MSCV++ will NOT inherently take .cc files. I am not certain this is still true in Visual Studio.net, so perhaps users of it can ignore this section. Lack of treating .cc files makes it difficult to compile GAMMA sources and programs since the .cc suffix is what has been traditionally used. Either all .cc files supplied with GAMMA must be changed to .cpp (including all references to .cc) OR Visual C++ must be taught how to deal with .cc files. We prefer the latter and luckily Microsoft has not left us in the dark on this issue. All one has to do is change the registry in a few places with regedit. This is discussed in the MS Knowledge Base, Article Q181506.

This is not very difficult to do, it will take you only 15 minutes (working slow.) If they haven't changed things at their site, this link will take you there directly: KB Article Q181506. If that does not work, go to the MS Knowledge Base Search. Just search in the area Visual C++ for a Specific article ID number with number Q181506. Follow the instructions found in that document and teach MSVC++ to understand what .cc files are. I have also put a local copy of this article here, hopefully fullfilling MS requirements to avoid any copyright infringements: Q181506 Local Copy.

A viable alternative would be to change all of the GAMMA .cc files to .cpp files. This is also easy to accomplish using any of several shareware programs that perform bulk file renaming. In contrast to use of the above change(s), which allows users to compile either .cc or .cpp files, this leaves MSVC++ with the inherent limitation of only recognizing .cpp files as containing valid C++ source code. Also, the provided workspaces for building GAMMA will no longer work without the user changing all of their file listings as well.

Setting MSVC++ Global Environment

 These MSVC++ workspace settings as are global, that is, they affect every project build in MSVC++. You may wish to record any changes if you need to set things back for other (non-GAMMA) project builds at a later time. The primary reason they may need adjustment is for differences in GAMMA installation directories as well as any changes made when building other MSVC++ projects. Note that the settings used here are adequate to build both GAMMA and PyGAMMA. Shown are those which differ from the MSVC++ defaults set when creating a typical project not using MFC.

MSVC++ Settings For GAMMA

  1.) Open GAMMA Project: Click on pygamma.dws icon
  2.) Open Options: Tools - Options
  3.) Select Tabs Tab: Set Tab Size 8, Indent Size 8, Keep Tabs
  4.) Select Directories Tab
  5.) Look at Include files: Show directories for: Include files
  6.) #Set GAMMA Headers: C:\GAMMA\GAMMA-4.1.0\SRC
  7.) *Set Python Headers: C:\PROGRAM FILES\PYTHON22\INCLUDE
  8.) *Set Boost.Python Headers: C:\PROGRAM FILES\BOOST\BOOST_1_28_0\
  9.) Look at Library files: Show directories for: Library files
 10.) Include GAMMA Library: C:\GAMMA\GAMMA-4.1.0\I686-PC-MSVC
 11.) *Include Python Library: C:\PROGRAM FILES\PYTHON22\LIBS
 12.) *Include Boost.Python Library:
e.g. C:\Program Files\Boost\boost_1_28_0\libs
\python\build\bin\boost_python.dll\msvc\release\runtime-link-dynamic
 13.) Close Tools
 
# Set this to where the sources of your version of GAMMA resides.
* The Python and Boost header and library includes are not needed unless one wishes to build PyGAMMA, the interactive version of GAMMA that runs under Python. Again, these must be set only if PyGAMMA is built. However, if they are present they will not adversely affect any other GAMMA builds.
