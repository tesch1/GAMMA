Microsoft
Visual C++
(Γ Version 4.1.0)
 
 
This directory exists in support of using Microsoft Visual C++ to build GAMMA and GAMMA based programs on Windows/Intel based computers. You MUST have a working version of MSVC++ on your computer system (PC) before these files will be of any use at all.
 

The figure at the left depicts a GAMMA directory structure, the directories of importance* to MSVC++ are highlighted in green. (Your directory structure may vary somewhat.) The directory i686-pc-msvc will contain any libraries and binary object files built under Visual C++. This will be created during the build of a GAMMA library if it does not currently exist. The directory src is where the GAMMA source codes are kept for a particular version (version 4.1.0 in the figure). It is important that both the GAMMA .cc and .h files are in that directory if you plan to build any GAMMA libraries under MSVC++. If you will use a pre-built library to build GAMMA based programs under MSVC++ then you need only the header (.h) files.

 
* PyGAMMA also uses directories that are part of Python & Boost, not part of GAMMA. Additionally, there are projects which compile and/or run example programs found in the demo and test directories.
 
As for the directory you are currently in, msvc, notice that there are a few sub-directories branching off of this location. The important ones are static, dynamic, and pygamma. Each of these contains an MSVC++ workspace that can be used to build the GAMMA library and/or use the GAMMA library to build and run demo, example, and test programs. As might be expected, there are very significant differences between their contents.

  • The i686-pc-msvc subdirectory contains any MSVC++ specific GAMMA binary files. This directory will be generated when a GAMMA library is built using the provided MSVC++ projects, or it will exist if pre-packaged MSVC++ libraries have been downloaded and installed.

  • The static subdirectory workspace contains a project to build GAMMA as a single static library. Additional projects in the workspace will build demo, test, and example programs using this static library. In most cases the GAMMA DLL is more preferable to use. It is funtionally the same as the static library, and any GAMMA program executables will be smaller. The one drawback in using the DLL is that GAMMA executables will not run unless the DLL is present.

  • The dynamic subdirectory workspace contains a project to build GAMMA as a single dynamic library (DLL). Additional projects in the workspace will build demo, test, and example programs using this DLL. This is usually what is built and used for GAMMA.

  • The pygamma subdirectory workspace contains a project to build PyGAMMA, the interactive version of GAMMA that runs under Python, as a single dynamic library (DLL).

  • The figs subdirectory contains images used in the display of this document. This should be of no concern to the user.

  • The src subdirectory contains GAMMA source code (.cc) files and GAMMA header (.h) files. Both source and header files are required in order to build any GAMMA library. Only the header files are required to build/run GAMMA based programs once the GAMMA library is available (built or downloaded.).

Prior to attempting any GAMMA library or GAMMA based program builds under MSVC++, it is ABSOLUTELY ESSENTIAL that you first configure MSVC++! Here are addiitonal GAMMA links on using MSVC++:
