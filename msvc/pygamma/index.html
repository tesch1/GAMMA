MSVC++ PyGAMMA
Build & Projects
(Γ Version 4.1.0)
 
 
This directory contains an MSVC++ workspace pygamma.dsw for the build of a Windows based PyGAMMA DLL as well as projects which demonstrate use of PyGAMMA. These notes were gleaned from my experience with MSVC++ version 6.0 adjusted to handle .cc files. You MUST have MSVC++ on your computer system (PC) before these files will be of any use to you.

The build of PyGAMMA is very similar to what is performed when GAMMA is built. The same GAMMA source codes are used in both. However PyGAMMA runs in a Python shell and its functions are exported into Python via Boost.Python. Thus, the build demands that both Python and Boost.Python have been installed on the system. Additonal information regarding PyGAMMA is available. See Python For Windows and See Boost.Python For MSVC++.

MSVC++ Workspace Configuration For GAMMA

The GAMMA supplied MSVC++ workspace/projects may fail if Visual C++ itself is not set up properly. This document will assume that you have already accomplished the global configuration, and need only perform any localized configuration required for individual projects. Each individual project will have its own particular settings, however most of these have already been set.

The following is a list of settings which the user should adjust if there are problems building the PyGAMMA DLL, its associated demo program, or any other GAMMA project in the workspace. The settings listed are those which differ from the MSVC++ defaults used when setting up a DLL project not using MFC. Again, let me emphasize that most of these settings should already be in place. The primary reason for listing them is so users can build their own PyGAMMA MSVC++ projects.

MSVC++ Project Settings For PyGAMMA

  1.) Open PyGAMMA Project: Click on pygamma.dws icon
 2.) Set Release Build: Upper Menu Bar Select Win32 Release
 3.) Open Project Settings: Right Click pygamma files - Settings...
 3.) Select Settings For: Win32 Release - C/C++
 4.) C/C++ Tab Project Options: Add /TP to list
 5.) Go To C/C++ Tab - Category: Preprocessor
 6.) Additional include directories: ./../src
 7.) Select Tab Link
 8.) Output file name:IC:\Program Files\Python22\DLLs\pygamma.dll
 9.) Close Project Settings

MSVC++ Build Of The PyGAMMA DLL

Do You Need To Build The DLL?

You may already have the PyGAMMA DLL! It may have come with your GAMMA distribution or perhaps you obtained it directly downloaded from a GAMMA WWW site. The PyGAMMA DLL is a single file, pygamma.dll. It is this file which supplies GAMMA functionality to the Python interpreter.

If you have a viable PyGAMMA DLL already you will have this file stored on your system. However, its physical location may vary with your setup. It may reside in the MSVC++ binaries directory. Alternatively, another likely location would be the Python DLL directory. Since (under Windows) if currently use Python 2.2 installed on my C drive in the Program Files folder, my PyGAMMA DLL is located in C:\Program Files\Python22\DLLs. If you have all of this DLL then you do not need to build it unless there is some unforseen troubles.

You Have Set Up MSVC++ For PyGAMMA Right?

The build of the PyGAMMA will fail if you have not properly set up MSVC++. There are global settings as well as project settings. Make sure both have been done.

You Have The GAMMA Sources Too?

You cannot build the PyGAMMA DLL without having the GAMMA sources, both the .cc and .h files. Have an look in the source directory to make sure there are lots of these in the sub-folders from this one.

Here is How To Build The PyGAMMA DLL

The primary project pygamma will build the PyGAMMA DLL from sources. The only important file created during the build is the DLL itself named pygamma.dll. It is this file that supplies GAMMA functions to the Python interpreter (as is used in ModuleDemos and ModuleTests). The build will produce the DLL pygamma.dll in the binary directory GAMMAVER/i686-pc-msvc. The DLL and any object files will all be placed into this directory. Note that if you clean the PyGAMMA DLL project all object files and the DLL will be deleted.

DLL Build Steps
 1.) Open MSVC++ Workspace: Click on pygamma.dsw icon [this dir]
 2.) Set pG project default: Project - Set Active Project - pygamma
 3.) Clean PyGAMMA project: Build - Clean
 4.) Build PyGAMMA Project: Build - Build pygamma.dll

Note that the PyGAMMA DLL will be named pygamma.dll. Furthermore, under MSVC++ if you clean the PyGAMMA library project all object files and the DLL file will be deleted.

MSVC++ Build & Run PyGAMMA Module Demos

GAMMA comes with a series of demonstration programs which will take the user through each module and detail what it provides and how it is used. The supplied MSVC++ workspace in this directory, pygamma.dsw contains a project pyModuleDemos for building and running these demos in a single program (as a single Python script). The demo runs interactively and will demand user input. Of course, you must have the PyGAMMA DLL on your system in order to run the demo program. The DLL is pygamma.dll and should reside in the Python DLL directory (or any directory searched by Python).

 1.) Open MSVC++ Workspace: Click on pygamma.dsw icon [this dir]
 2.) Demo Project Default: Project - Set Active Project - pyModuleDemos
 3.) Clean Demo Project: Build - Clean
 4.) Build Demo Project: Build - Build
 5.) Run Demo Project: Build - ! Execute python.exe

MSVC++ GAMMA Testing Build/Run Using Static Library

GAMMA comes with a series of testing programs which will attempt to verify that GAMMA is working properly. The supplied MSVC++ workspace in this directory, gamma.dsw contains a project testmodules for building and running these tests in a single program. It will produce an executable TestGAMMA.exe in this directory, then run it. The test program will link to the static library and runs interactively. It will demand some user input. Of course, you must have the GAMMA static library and the GAMMA headers on your system in order to build and run the test program. The library is gamma.lib and should reside in the Visual C++ binary directory ../../i686-pc-msvc. The headers will be the .h files found in the folders stemming from the source directory ../../src.

 1.) Open MSVC++ Workspace: Click on gamma.dsw icon [this dir]
 2.) Temo Project Default: Project - Set Active Project - ModuleTests
 3.) Clean Test Project: Build - Clean
 4.) Build Test Project: Build - Build TestGAMMA.exe
 5.) Run Test Project: Build - ! Execute TestGamma.exe

MSVC++ PyGAMMA DLL Cleanup

If you have built the PyGAMMA library there will be many binary object files placed into the MSVC++ binary directory of GAMMA, ../../i686-pc-msvc. These are not needed to build (Py)GAMMA programs, only the DLL is needed for PyGAMMA. Sou may consider whether the object files should be removed in order to save disk space. In addition, if you have run the demo program then there will be many files cluttering up this directory that were produced when the demo ran. In order to quickly remove these there is a batch file provided, clean.bat.

If you click (from Windows) the icon for clean.bat it will remove any files produced from running the tests and demos, including the executables (which are easily remade). It will also remove the object files in the MSVC++ binary directory of GAMMA. It will NOT remove the static PyGAMMA DLL. In short, running clean.bat will take away all files that did not come with the GAMMA distribution EXCEPT the PyGAMMA DLL. Recall that using the MSVC++ clean function for the DLL build will delete the DLL and object files, which is why it is probably best to use clean.bat after the build is done if you intend to keep the DLL.
