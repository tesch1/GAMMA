MSVC++ Static Lib.
Build & Projects
(Γ Version 4.1.0)
 
 

This directory contains an MSVC++ workspace gamma.dsw which will build a Windows based GAMMA static library as well as smaller projects which will compile/run demonstration and testing programs which use the static library. These notes were gleaned from my experience with MSVC++ version 6.0 adjusted to handle .cc files. You MUST have MSVC++ on your computer system (PC) before these files will be of any use to you.

MSVC++ Projects In Workspace gamma.dsw
NameDescription
gammaBuilds GAMMA as a Static Library
ModuleDemos  Compiles & Runs GAMMA demo program
ModuleTests  Compiles & Runs GAMMA test program

Recommended Steps

  1. Configure MSVC++ For GAMMA Project Builds
  2. Build The GAMMA Static Library (if needed)
  3. Build/Run GAMMA Module Demos
  4. Build/Run GAMMA Module Tests
  5. Clean Out Any Unneeded Files From Builds (if desired)

MSVC++ Workspace Configuration For GAMMA

The GAMMA supplied MSVC++ workspace/projects may fail if Visual C++ itself is not set up properly. This document will assume that you have already accomplished the global configuration, and need only perform any localized configuration required for individual projects. Each individual project will have its own particular settings, however most of these have already been set.

The following is a list of settings which the user should adjust if there are problems building the GAMMA static library, demo program, test program, or any other GAMMA project in the workspace. The settings listed are those which differ from the MSVC++ defaults used when setting up a static library project not using MFC. Again, let me emphasize that most of these settings should already be in place. The primary reason for listing them is so users can build their own GAMMA MSVC++ projects.

MSVC++ Project Settings For GAMMA

  1.) Open GAMMA Project: Click on gamma.dsw icon
 2.) Set Release Build: Upper Menu Bar Select Win32 Release
 3.) Open Project Settings: Right Click gamma files - Settings...
 4.) Select Settings For: Win32 Release - C/C++
 5.) C/C++ Tab Project Options: Add /TP /Zm200 to list
 6.) Select Tab Link
 7.) Output file name:I../../i686-pc-msvc\gamma.lib
 8.) Close Project Settings

MSVC++ Build Of The GAMMA Static Library

Do You Need To Build The Library?

You may already have the static library! It may have come with your GAMMA distribution or it you may have directly downloaded it from a GAMMA WWW site. The GAMMA static library is a single file, gamma.lib. It is this file to which users link to when building GAMMA based programs using the static library.

If you have a viable GAMMA static library already you will have this file as well as all the GAMMA headers. Check to see if the headers exist in the folders stemming from the source directory (scads of .h files). Check to see if you have the static library in the MSVC++ binaries directory. If you have all of this (static library, GAMMA headers) then you do not need to build it unless there is some unforseen troubles.

You Have Set Up MSVC++ For GAMMA Right?

The build of the static library will fail if you have not properly set up MSVC++. There are global settings as well as project settings. Make sure both have been done.

You Have The GAMMA Sources Too?

You cannot build the static library without having the GAMMA sources, both the .cc and .h files. Have an look in the source directory to make sure there are lots of these in the sub-folders from this one.

Here is How To Build The Static Library

The primary project gamma will build GAMMA from sources as a statically linked library. The only important file created during the build is the library itself named gamma.lib. It is theis file that GAMMA programs (such as those in ModuleDemos and ModuleTests) actually link to. The build will produce the static library gamma.lib in the binary directory GAMMAVER/i686-pc-msvc. The static library and object files will all be placed into this directory. Note that if you clean the GAMMA library project all object files and the library will be deleted.

Static Library Build Steps
 1.) Open MSVC++ Workspace: Click on gamma.dsw icon [this dir]
 2.) Set GAMMA project default: Project - Set Active Project - Gamma
 3.) Clean GAMMA project: Build - Clean
 4.) Build GAMMA Project: Build - Build gamma.lib

Note that the static GAMMA library will be named gamma.lib. This is in contrast to the Unix library naming convention which would use libgamma.a. Furthermore, under MSVC++ if you clean the GAMMA library project all object files and the library file will be deleted.

MSVC++ Build & Run GAMMA Module Demos

GAMMA comes with a series of demonstration programs which will take the user through each module and detail what it provides and how it is used. The supplied MSVC++ workspace in this directory, gamma.dsw contains a project ModuleDemos for building and running these demos in a single program. It will produce an executable DemoGAMMA.exe in this directory, then run it. The demo runs interactively and will demand user input. Of course, you must have the GAMMA static library and the GAMMA headers on your system in order to build and run the demo program. The library is gamma.lib and should reside in the Visual C++ binary directory ../../i686-pc-msvc. The headers will be the .h files found in the folders stemming from the source directory ../../src.

 1.) Open MSVC++ Workspace: Click on gamma.dsw icon [this dir]
 2.) Demo Project Default: Project - Set Active Project - ModuleDemos
 3.) Clean Demo Project: Build - Clean
 4.) Build Demo Project: Build - Build DemoGAMMA.exe
 5.) Run Demo Project: Build - ! Execute DemoGamma.exe

MSVC++ GAMMA Testing Build/Run Using Static Library

GAMMA comes with a series of testing programs which will attempt to verify that GAMMA is working properly. The supplied MSVC++ workspace in this directory, gamma.dsw contains a project testmodules for building and running these tests in a single program. It will produce an executable TestGAMMA.exe in this directory, then run it. The test program will link to the static library and runs interactively. It will demand some user input. Of course, you must have the GAMMA static library and the GAMMA headers on your system in order to build and run the test program. The library is gamma.lib and should reside in the Visual C++ binary directory ../../i686-pc-msvc. The headers will be the .h files found in the folders stemming from the source directory ../../src.

 1.) Open MSVC++ Workspace: Click on gamma.dsw icon [this dir]
 2.) Temo Project Default: Project - Set Active Project - ModuleTests
 3.) Clean Test Project: Build - Clean
 4.) Build Test Project: Build - Build TestGAMMA.exe
 5.) Run Test Project: Build - ! Execute TestGamma.exe

MSVC++ GAMMA Static Library Cleanup

If you have built the GAMMA library there will be many binary object files placed into the MSVC++ binary directory of GAMMA, ../../i686-pc-msvc. These are not needed to build GAMMA programs, only the library is needed, so you may consider that the object files should be removed in order to save disk space. In addition, if you have run the demo program then there will be many files cluttering up this directory that were produced when the demo ran. In order to quickly remove these there is a batch file provided, clean.bat.

If you click (from Windows) the icon for clean.bat it will remove any files produced from running the tests and demos, including the executables (which are easily remade). It will also remove the object files in the MSVC++ binary directory of GAMMA. It will NOT remove the static GAMMA library. In short, running clean.bat will take away all files that did not come with the GAMMA distribution EXCEPT the static GAMMA library. Recall that using the MSVC++ clean function for the static library build will delete the library and object files, which is why it is probably best to use clean.bat after the build is done if you intend to keep the library.
