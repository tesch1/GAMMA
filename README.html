Builds of GAMMA (Currently) Vary From The Normal Steps

Rather than follow the steps outlined later in this file (left in for when the Makefile is fixed), please do the following instead...

For Windows/Cygwin

There is a makefile provided to build GAMMA as a DLL and/or as a static library. Simply look in the directory primal/CygWin and follow the instructions provided. In this case one will typically build GAMMA as a DLL with the -mno-cygwin flag. Note that the DLL has been prebuilt and may be easily downloaded and installed, .... so you should have good reason to do this

For Windows/MinGW

There is a makefile provided to build GAMMA as a DLL and/or as a static library. Simply look in the directory primal/MinGW and follow the instructions provided. In this case one will typically build GAMMA as a DLL. Note that the DLL has been prebuilt and may be easily downloaded and installed, .... so you should have good reason to do this build.

For Windows/MSVC++

There are workspaces provided for the build of GAMMA as either a static library or a DLL. Look in the directory msvc and follow the instructions provided. If no such file/directory exists, then download them at any GAMMA WWW site. In this case one will typically build GAMMA as a DLL. Note that the DLL has been prebuilt and may be easily downloaded and installed, .... so you should have good reason to do this build.

For Linux/GCC

There is a makefile to build GAMMA as a shared object library. Look in the directory primal/Linux and follow the instructions provided. Note that the shared library has been prebuilt and may be easily downloaded and installed (using RPM if desired), .... so you should have good reason to do this build.

For OSX/GCC

There is a makefile to build GAMMA as a shared object library. Look in the directory primal/OSX and follow the instructions provided. Note that the shared library has been prebuilt and may be easily downloaded and installed (using RPM if desired), .... so you should have good reason to do this build.

Other OS/Compiler

Sorry, you will have to do a bit more work to get GAMMA built. What you must do is copy the primal Makefile that most closely resembles your OS/Compiler combination and modify it to work. For example, suppose one were building under IRIX on an SGI. I would copy the directory primal/Linux to a new directory primal/Irix. Then I would edit the makefile in the directory primal/Irix to set the directories, compiler flags, etc to match the demands of IRIX and the compiler to be used.

If you need help with doing this send some e-mail to gamma@magnet.fsu.edu. Often Scott will just log in remotely to your system and do the build for you, in return he will post the binaries and makefiles so that others can use them.

    Quick Build & Install (Unix/Linux/CygWin)
    In a (bash) shell type:
  1. ./configure
    		2.   - Set up for system specifics
  2. make
    		3.   - Compile GAMMA source codes
  3. make install
    		4.   - Copy GAMMA script to accessible location

Explicit installation instructions may be found at the GAMMA WWW site, (or, if it has been installed, in your local documentation directory doc sub-directory). Should that prove insufficient, you may Request Help Directly. Common problems and suggested solutions are shown below:

Failed Step What To Do
./configure Fails   If a permission denied error is seen then you must set the configure file to be executable: chmod a+x configure, then reissue the configuration command, ./configure.
Occasionally configuration failure is caused by residual Windows linefeed characters (^M) in the configuration and/or makefiles. Run the command ./DOSNoMore then reissue the configuration command, ./configure.
make Fails   Make sure you are using the Gnu make program! GAMMA has complicated Makefiles that use commands that some make programs simply do not understand. Type make -v (or gmake -v) to see what version of Gnu make you are using.
Make sure that your C/C++ compiler is working. Try and compile some simple C (file.c) and C++ (file.cc or file.cpp) programs. Recent versions of Gnu GCC (>= 2.8.x) usually work just fine building GAMMA. With GCC you can use gcc -v to see the compiler version you are using.
If the above are fine, it's probably best to just fill out a report of the failure. Include as much information as you can and send in the specific lines where the compilation failed. You might even report any warnings. Although non-fatal, they may indicate some real problem. B
make install Fails   Simply copy the produced script, gamma, into any directory contained in the user(s) path.

The subdirectory doc is set to contain local GAMMA documentation in html and PDF. The subdirectory demo is set to contain some quick examples to insure GAMMA is working properly. If these are not present on your system you may wish to download them and other GAMMA documentation.

GAMMA is a C++ library expressly for simulation of Magnetic Resonance experiments. It was written by Scott A. Smith and Tilo Levante under the guideance of B.H. Meier and R.R. Ernst at the ETH in Zürich, as described in the article
"Computer Simulations in Magnetic Resonance.
An Object Oriented Programming Approach",
S.A. Smith, T.O. Levante, B.H. Meier, and R.R. Ernst,
J. Magn. Reson., 106a, 75-105, (1994).
It has been under the administration of Dr. Smith for the past several years and currently resides at the NHMFL. Proper reference should be given to the citation above when GAMMA simulations are utilized in papers and/or presentations (& perhaps a thank you to the NHMFL for continuing its support).
